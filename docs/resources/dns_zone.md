# f5xc_dns_zone (Resource)

Manages a DNSZone resource in F5 Distributed Cloud.

~> **Note** For more information about this resource, please refer to the [F5 XC API Documentation](https://docs.cloud.f5.com/docs/api/).

## Example Usage

```terraform
# Dns Zone Resource Example
# Manages a DNSZone resource in F5 Distributed Cloud.

# Basic Dns Zone configuration
resource "f5xc_dns_zone" "example" {
  name      = "example-dns-zone"
  namespace = "system"

  labels = {
    environment = "production"
    managed_by  = "terraform"
  }

  annotations = {
    "owner" = "platform-team"
  }

  # DNS Zone configuration
  # Primary DNS zone
  primary {
    soa_record_parameters {
      refresh = 86400
      retry   = 7200
      expire  = 3600000
      ttl     = 86400
      neg_ttl = 1800
    }
    default_rr_set_group {}
    default_soa_parameters {}
    dnssec_mode {
      disable {}
    }
  }
}
```

### Nested Schema for `soa_parameters`

Optional:

- `expire` (Number) Expire. expire value indicates when secondary nameservers should stop answering request for this zone if primary does not respond
- `negative_ttl` (Number) Negative TTL. negative ttl value indicates how long to cache non-existent resource record for this zone
- `refresh` (Number) Refresh interval. refresh value indicates when secondary nameservers should query for the SOA record to detect zone changes
- `retry` (Number) Retry Interval. retry value indicates when secondary nameservers should retry to request the serial number if primary does not respond
- `ttl` (Number) TTL. SOA record time to live (in seconds)

Valid time units are "s" (seconds), "m" (minutes), "h" (hours). Read operations occur during any refresh or planning operation when refresh is enabled.
